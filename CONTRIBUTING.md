# Contributing to API Posts Block

Thank you for your interest in contributing! Here are some guidelines:

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/adamm13/api-posts-block.git`
3. Create a feature branch: `git checkout -b feature/your-feature`
4. Install dependencies: `npm install`

## Development Workflow

1. Make your changes in the `src/` directory
2. Test your changes using `npm run dev`
3. Build your changes: `npm run build`
4. Commit with clear messages: `git commit -m "Add feature description"`
5. Push to your fork and submit a Pull Request

## Code Standards

- Follow WordPress coding standards
- Use ESLint and Stylelint: `npm run lint:js` and `npm run lint:style`
- Format code: `npm run format`
- Add comments for complex logic
- Test on multiple WordPress versions

## Reporting Issues

- Use the GitHub Issues tracker
- Include WordPress version, PHP version, and browser information
- Provide clear steps to reproduce
- Include error logs if applicable
